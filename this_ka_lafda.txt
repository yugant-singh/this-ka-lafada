ðŸ“Œ Function Methods in JavaScript: call, apply, bind --->
-------------------------------------------------------------------------------------------------
1. call()
ðŸ”¹ Definition:

call() ==> ek function method hai jo aapko allow karta hai ki aap function ke andar this ko manually set karo.

 
 * Function turant execute hota hai.
 * Arguments comma separated pass hote hain.

ðŸ”¹ Example:
function greet(city, country) {
  console.log(`Hello, I am ${this.name} from ${city}, ${country}`);
}

const person1 = { name: "Ali" };
const person2 = { name: "Sara" };

// Using call()
greet.call(person1, "Lahore", "Pakistan");
// Output: Hello, I am Ali from Lahore, Pakistan

greet.call(person2, "Delhi", "India");
// Output: Hello, I am Sara from Delhi, India

-------------------------------------------------------------------------------------------------
2. apply()
ðŸ”¹ Definition:

* apply() ka kaam call() jaisa hi hota hai, difference sirf arguments dene ka hai.
* Isme arguments array ([]) ke form mein pass hote hain.
* Function turant execute hota hai.

ðŸ”¹ Example:
greet.apply(person1, ["Karachi", "Pakistan"]);
// Output: Hello, I am Ali from Karachi, Pakistan

greet.apply(person2, ["Mumbai", "India"]);
// Output: Hello, I am Sara from Mumbai, India


ðŸ‘‰ Use case: Jab arguments already ek array ke form mein available ho.

const details = ["New York", "USA"];
greet.apply(person2, details);
// Output: Hello, I am Sara from New York, USA
-------------------------------------------------------------------------------------------------
3. bind()
ðŸ”¹ Definition:
* bind() ek naya function return karta hai jisme this permanently set hota hai.
* Function turant execute nahi hota, aapko manually call karna padta hai.
* Arguments comma separated pass hote hain (same as call).

ðŸ”¹ Example:
const boundGreet = greet.bind(person1, "Islamabad", "Pakistan");

// function abhi execute nahi hua
boundGreet(); 
// Output: Hello, I am Ali from Islamabad, Pakistan

ðŸ‘‰ Use case: Jab aapko ek function future mein use karna ho jisme this fix ho.

ðŸ“Œ Q5
const button = {
  text: "Submit",
  click: function () {
    console.log(this.text + " clicked");
  },
};

setTimeout(button.click, 1000);       // â“ yahan output kya hoga?
setTimeout(button.click.bind(button), 2000); // â“ aur yahan kya hoga?
-------------------------------------------------------------------------------------------------


ðŸ“– Ab iska detail explanation:
1ï¸âƒ£ setTimeout(button.click, 1000);

Tumne button.click ko reference ke form me pass kar diya setTimeout ko.

Jab 1 second baad ye function execute hota hai, tab wo independent function ban jata hai (uska this context chala jata hai).

Default case me this = window (browser) / global (Node.js).

window ya global object ke andar koi text property nahi hai â†’ isiliye output:

undefined clicked

2ï¸âƒ£ setTimeout(button.click.bind(button), 2000);

Yahan tumne bind(button) use kiya.
bind() ek naya function return karta hai jisme this permanently button object pe set hota hai.
Ab jab 2 second baad function run hota hai, this.text = "Submit" use hota hai.
Output:

Submit clicked

âœ… Takeaway

Jab tum function ko directly reference ke form me pass karte ho (e.g. setTimeout ya event handler me), tab this ka context lost ho jata hai.

bind() ka use karke tum this ko fix kar sakte ho.
